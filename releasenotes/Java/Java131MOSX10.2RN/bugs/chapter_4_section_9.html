Java Graphics

Radar #2776033

Batik: Warnings printed out when opening samples.

Description:

Batik throws a number of exceptions when opening the sample .svg files with SVGBrowser.

Workaround:

This has been fixed.


Radar #2805700

OptimizeIt starts responding slowly when moved to a second monitor.

Description:

While running OptimizeIt, the application starts drawing slowly when it's moved to a second monitor. When moved back to the original monitor, it draws as it did before being moved.

Workaround:

This has been fixed.


Radar #2822976

Can't print java.awt.List.

Description:

Images of a java.awt.List may not be printed.

Workaround:

None.


Radar #2869569

Problems reported with repainting in the 1.3.1 Java update 1.

Description:

It has been reported that in some applications that lightweight components do not draw backgrounds, as their update method has been overwritten. They still, however, paint their background. This did not happen prior to the Java update and works in some situations. For example, when switching to another window the window will be redrawn correctly.

Workaround:

None at this time.


Radar #2869658

AWT components' attributes aren't drawn any more.

Description:

As of Java 1.3.1 Update 1, background and foreground colors on AWT components are not displayed any more.

Workaround:

This has been fixed.


Radar #2879494

Incorrect drawing of polylines under hardware acceleration.

Description:

When a Java program is run with and without hardware acceleration, polylines are incorrectly drawn in the hardware-accelerated version, but appear as intended in the non-accelerated version.

Workaround:

This has been fixed.


Radar #2880308

XOR fails with hwaccel on.

Description:

There is a bug that affects XOR drawing when hwaccel is turned on, affecting simple graphical primitives, such as lines.

Workaround:

Turn hwaccel off if your code uses XOR painting mode.


Radar #2891306

In the Mac OS X Java 1.3.1 Update 1 graphics.drawImage(...) fails to draw color images.

Description:

After installing the Mac OS X Java 1.3.1 Update 1, drawing an image with both source and destination rectangles specified failed in some cases. These routines had worked prior to the update.

Workaround:

This has been fixed.


Radar #2915929

BGR Image types have colors inverted.

Description:

BGR Image Types have several known problems, generally related to problems in GL hardware acceleration.

Workaround:

Use the native RGB Image Types. This has the added benefit of being much faster. When this is not a feasible solution, turn off hardware acceleration for the Java application experiencing the problem. This typically has the unfortunate, but expected, consequence of slower graphics performance.


Radar #2918504

Java applications suffer severe performance slowdown when their windows are under the Dock.

Description:

There are certain system configurations where Java applications will suffer severe performance slowdown when any of their windows are placed under the Dock.

Workaround:

If your Java application creates a window that is large enough to have the Dock show on top of it, you might do one of the following two things to avoid a slowdown in performance:

  1. Hide the Dock.

  2. Resize the window so that it's not under the Dock.

  3. Turn off hwaccel for that Java application for that particular configuration (see What's New in Java for Mac OS X Version 10.2 for more information).
