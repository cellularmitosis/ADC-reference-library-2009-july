Reusable Components

A reusable component is a component that you can include in another component. For example, you can define a reusable component that's a toolbar used in many other components.

Another type of component is one that is wrapped around other content. For example, it could contain a toolbar and space for banner ads. This type of reusable component contains a WOComponentContent element, which is where the other content is placed.

To make a component into a reusable component:

  • In the component's Inspector, choose whether this is a Full Document or a Partial Document.

    Generally, if this reusable component will be fully contained within other components, select Partial Document. And if this reusable component contains other components by using WOComponentContent, select Full Document.

    Note that if the reusable component needs to define an attribute that's part of the <BODY> tag, such as page color or title, it must be a Full Document and any component that uses it must be a Partial Document.

Choosing Attributes for a Component

A reusable component's attributes are like a function's arguments: They let one component send data to a reusable component within it.

To edit a component's attributes

  • Click the API button , click the Bindings tab, and Control-click in the bindings table, and use the menu that appears.

To require a user to bind an attribute:

  • Click in the Required column.

    WebObjects Builder creates a validation rule that raises a warning if the attribute is not bound.

Your component may need to be able to change an attribute's value. For example, the attribute could hold the value of a form element.To ensure that a component can change an attribute's value:

  • Click in the Will Set column.

    WebObjects Builder creates a validation rule that raises a warning if the user assigns a constant value to this attribute.

You can give the user the option of choosing an attribute's value from a menu of possible values, such as common date formats, MIME types, and the names of the pages in the current project. To let the user choose an attribute's value from a menu of values:

  • Use the Value Sets pop-up menu.

Choosing How to Validate a Component

Validation rules help to ensure that a user binds all the component's attributes correctly. For example, a component may have two similar attributes, and the user should set only one of them. (much like a dynamic image which can have either a filename or a src but not both).

For simple rules, you can use the Required and Will Set columns in the Bindings section of the API window. The Required column ensures that the user binds the attribute, and the Will Set column ensures that the user does not bind the attribute to a constant value.

When you need more complex rules, use the Validation tab of the API window. To add a validation rule:

  • In the API editor, click the Validation tab, Control-click in the Messages list, and select Add Rule.

Each rule consists of a message and an error condition. The message is the text the user will see if the error condition is true. To edit that text, double-click it.

The Error Condition is a boolean expression. Each line of that condition specifies an attribute and whether that attribute is bound, unbound, settable, or unsettable. (An attribute is unsettable if it is bound to a constant value.)

To edit an existing rule:

  • Click its message in the Messages list.

To add another clause to a condition:

  • Press And or Or.

    The selected condition is combined by AND or OR to the new condition.

To negate a condition or a group of conditions combined by AND or OR:

  • Select it and press Not

To remove a condition or group of conditions:

  • Select it and press Remove.

Choosing How to Display the Component

A user can view documentation for a component by pressing the Book button in the component's Inspector. To specify what documentation to display:

  • In the API editor, click the Display tab, and use the Documentation section.

When a user drags a reusable component to a page, WebObjects Builder displays the reusable component's HTML by default. If the component's purpose isn't obvious from that HTML, you can specify a proxy image to be displayed instead. To specify a proxy image:

  • In the API editor, click the Display tab, and use the Image section.
