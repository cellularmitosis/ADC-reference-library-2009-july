Radar # 2510039

Add get/setCreationDate methods to MRJFileUtils.

Description:

There is currently no way to access the creation date of a file without going to native code or JDirect. MRJFileUtils already includes a setModified method; it should also include getCreationDate and setCreationDate methods.

Workaround:

None.


Radar # 2585075

MRJFileUtils.setDefaultFileType, setDefaultFileCreator don't work in Mac OS X.

Description:

Certain functions present in Mac OS 9’s Java implementation no longer work in Mac OS X.

Workaround:

In Mac OS X, applications behavior should not be dictated by a file type code only. These methods depend on a certain filesystem, HFS(+). Since Mac OS X can run on other file systems, these methods should not be used.
