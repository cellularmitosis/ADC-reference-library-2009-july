Radar #2680426

handleQuit is called twice if there is also an MRJAboutHandler.

Description:

Code that registers both an MRJAboutHandler and an MRJQuitHandler uses a separate thread to display a Save Changes dialog from its handleQuit method.

Workaround:

Use a boolean to find out whether or not handleQuit() has already been called:

Boolean alreadyQuitting = false;
public void handleQuit() {
    if (!alreadyQuitting) {
        alreadyQuitting = true;
        // do quit handling here...
    }
}


Radar #2720352

MRJOpenDocumentHandler not called if an application was launched by double-clicking a document in the Finder.

Description:

Native Mac OS X applications claim certain file types in the system. If a file of that type is double-clicked, the appropriate application will be launched (if not already running) and sent an AppleEvent to open that file. Java applications use the MRJOpenDocumentHandler to receive that AppleEvent. Double-clicking on a file that belongs to a Java application will launch the application if it is not running, but the initial AppleEvent will be lost. If the application is already running when a document is double-clicked in Finder, the AppleEvent will be handled properly, calling the MRJOpenDocumentHandler.

Workaround:

There currently is none.

Workaround:

This is fixed in the current release.


Radar #2889765

MRJ DnD is deprecated in Mac OS X 10.2.

Description:

Apple had provided an extension to the JDK called MRJ DnD. As of Mac OS X 10.2, MRJ DnD is deprecated and replaced by Java2 DnD.

Workaround:

Use Java2 DnD in place of MRJ DnD.


Radar #2925661

APIs have been deprecated in this release.

Description:

Several APIs have been deprecated for this release. Refer to the Java 2 Platform, Standard Edition, v 1.3.1 API Specification located at /Developer/Documentation/Java/Reference/docs/api/index.html for further details.

The following calls have been deprecated and will not be replaced:

com.apple.mrj.MRJFileUtils.setDefaultFileType
com.apple.mrj.MRJFileUtils.setDefaultFileCreator
com.apple.mrj.MRJFileUtils.findApplication

The following calls are deprecated and have replacements:

com.apple.mrj.MRJMenuUtils.setMenuItemCmdKey
com.apple.mrj.MRJFileUtils.setFileLastModified

These are deprecated and have partial replacements:

com.apple.mrj.datatransfer.*
com.apple.mrj.dnd.*
com.apple.mrj.console.*

Workaround:

Use the available replacements as documented in the Java 2 Platform, Standard Edition, v 1.3.1 API Specification located at /Developer/Documentation/Java/Reference/docs/api/index.html.


Radar #2956026

MRJQuitHandler behaves differently in Mac OS X 10.2.

Description:

Previously, clients had to call explicitly System.exit() from the quit handler if termination was desired. Now the quit handler calls it by default unless the quit handler throws an exception to prevent it.

Workaround:

Not applicable, as MRJQuitHandler behaves correctly.
