Validating Documents

The Document Type Definition (DTD)for the Final Cut Pro XML Interchange Format lets you independently validate an interchange format document before you import it into Final Cut Pro.

Note: In Final Cut Pro, the DTD for the interchange format is not integrated into the import process. That is, Final Cut Pro does not validate a file against the DTD during import.

There are four separate DTDs for versions 1, 2, 3, and 4 of the interchange format.

In this section:

Using the DTDs
DTD for Version 1 of the Interchange Format
DTD for Version 2 of the Interchange Format
DTD for Version 3 of the Interchange Format
DTD for Version 4 of the Interchange Format


Using the DTDs

The DTDs for the Final Cut Pro XML Interchange Format are available in text file format. You can find these files on the Apple Developer web site at the following URL: http://developer.apple.com/appleapplications.

You can use these DTDs to validate an interchange format document prior to importing the document into Final Cut Pro. (Note that validation against the DTD does not happen automatically during import.)

You can use the xmllint program available in Mac OS X version 10.3 to validate an interchange format document. Follow these steps:

  1. Place a copy of the DTD file in the same folder with the interchange format document you want to validate.

  2. Open the Terminal application and navigate to the directory with the DTD file.

  3. Execute the following command line: %xmllint --noout --dtdvalid fcpxmlv1.dtd myxmlfile.file where myxmlfile.xml is the name of the file you want to validate
