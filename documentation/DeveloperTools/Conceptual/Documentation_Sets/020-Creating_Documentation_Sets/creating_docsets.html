Creating Documentation Sets

A documentation set is a standard Mac OS X bundle that packages a set of HTML (or PDF) documentation files, along with the information needed to access and display that documentation in the Xcode Documentation window. This chapter describes the directory hierarchy, naming conventions, and file-system locations of a documentation set bundle.

Organizing Documentation Set Bundles

The first step in creating a documentation set is to create and populate the folder hierarchy of the documentation set bundle. The documentation set structure that you create should look similar to that shown in Figure 2-1.

You must place all of a documentation set’s documentation files inside the Resources/Documents directory.

Within the Documents directory, you can use any arbitrary hierarchy to organize your documentation files. In this example, the documentation set contains a Tutorial.html file and a directory of additional documentation files, UserGuide.

Xcode and docsetutil support both HTML and PDF documentation files. Your documentation set can contain files in either format.

The Documents directory, like any other bundle resource, can be localized. For the structure of a localized documentation set bundle, see “Localizing a Documentation Set Bundle.”

Naming Documentation Sets

Because documentation sets are installed in standard locations, documentation set bundle names should be chosen to minimize the chance of conflicts. You should name documentation sets using the uniform type identifier (UTI)–style used for the CFBundleIdentifier property, as described in “Property List Key Reference.”

Documentation set bundle names must include the docset extension. So, for example, a documentation set bundle name may look like this:

com.mycompany.MyDocSet.docset

Choosing Documentation Set Locations

When it first displays the Documentation window, Xcode scans for all installed documentation sets. It searches for these documentation sets in the following locations, listed in the order in which they are searched:

  1. <Xcode>. Documentation sets are located at <Xcode>/Documentation/DocSets.

    In Xcode 3.0 users can install multiple versions of Xcode on a single computer. Xcode always searches for documentation sets within the <Xcode> directory in which the active Xcode binary resides. For example, if the user has installed Xcode 3.0 at /Xcode 3.0, Xcode searches for documentation sets at /Xcode 3.0/Documentation/DocSets..

  2. ~/Library/Developer/Shared/Documentation/DocSets. This location contains documentation sets specific to the user that is currently logged in to the system.

  3. /Library/Developer/Shared/Documentation/DocSets. This location contains documentation sets viewable by all users of the current system.

  4. /Network/Library/Developer/Shared/Documentation/DocSets. This location contains documentation sets used by a group of people on the same network.

  5. /System/Library/Developer/Shared/Documentation/DocSets. You should not install files in the system domain.

In addition to these standard locations, users can add their own locations to the list of documentation set locations in Xcode Preferences > Documentation. With the exception of the Xcode directory, which is always searched first, users can reorder the Documentation Set Locations list. Xcode searches for documentation sets at both standard and user-defined locations, in the order in which they appear in this list.

If the same documentation set appears in more than one location, Xcode uses the first instance that it finds. For more on documentation set locations, see Xcode User Guide.

You should install your documentation set in the same file system domain as the software product that it documents. For example, if your product is installed in the local domain—say, at /Applications/MyProduct—your documentation set should be installed at /Library/Developer/Documentation/DocSets.

In some cases, you may wish to install your documentation set in a location other than the standard documentation set locations. You can still make sure that Xcode finds and accesses your documentation set by creating a symbolic link (symlink) to your documentation set and placing that symlink in one of the standard documentation locations.

After the initial installation, users can move any documentation set—with the exception of the Developer Tools documentation set provided by Apple—to an arbitrary file-system location. Unless they move the documentation set to one of the standard locations, however, users must also ensure that the new location appears in the documentation set locations in Xcode Preferences > Documentation.

As a documentation set provider, you do not need to worry about the user relocating your documentation set. As long as the user adds the new location of their documentation directory as a documentation set location, Xcode will continue to be able to locate your documentation.
