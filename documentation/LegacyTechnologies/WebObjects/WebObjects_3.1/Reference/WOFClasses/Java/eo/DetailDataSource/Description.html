Class next.eo.DetailDataSource

CLASS DESCRIPTION

Extends:
next.eo.DataSource

DetailDataSource defines a data source for use in master-detail configurations, where operations in the detail data source are applied directly to properties of a master object.

To set up a DetailDataSource programmatically, you typically create it by sending a dataSourceQualifiedByKey message to the master data source, then establish the master object with a qualifyWithRelationshipKeyAndObject message.

CONSTRUCTORS

DetailDataSource

public DetailDataSource()
public DetailDataSource(next.eo.DataSource masterDataSource, java.lang.String relationshipKey)

Returns a newly-allocated and initialized DetailDataSource object. If the parameters are supplied, this method initializes a newly allocated DetailDataSource to provide objects based on a relationship of objects in masterDataSource named by relationshipKey. The receiver initially has no master object selected; to select one, use qualifyWithRelationshipKeyAndObject.This is the designated initializer for the DetailDataSource class.

METHODS

detailKey

public java.lang.String detailKey()

Returns the name of the relationship for which the receiver provides objects, as set in either the constructor or qualifyWithRelationshipKeyAndObject. If none has been set yet, returns nil.

masterDataSource

public next.eo.DataSource masterDataSource()

Returns the receiver's master data source.

masterObject

public java.lang.Object masterObject()

Returns the object in the master data source for which the receiver provides objects. You can change this with a qualifyWithRelationshipKeyAndObject message.

qualifiyWithRelationshipKeyAndObject

public void qualifiyWithRelationshipKeyAndObject(java.lang.String relationshipKey, java.lang.Object masterObject)

Configures the receiver to provide objects based on the relationship of masterObject named by relationshipKey. relationshipKey can be different from the one used with the constructor, which changes the relationship for the receiver. If masterObject is nil, this method causes the receiver to return an empty array when sent a fetchObjects message.
