Converting Projects From Earlier Releases

WebObjects applications from a prior release cannot simply be recompiled using WebObjects 4.5 Developer; you must first make a number of changes to your project's source code. To aid in this process, WebObjects 4.5 Developer includes a set of "tops" scripts that automatically make many of the changes for you.

If you are converting an application created with WebObjects 4.0 (or 4.01), you need only rebuild your app using WebObjects 4.5--unless your app uses Direct To Web, in which case you'll need to read " Converting Projects From WebObjects 4 ," below. If your WebObjects applications were created with WebObjects 3.5, you'll first need to perform those steps outlined in the sections beginning with Converting Projects from WebObjects 3.5 to 4.0 . Then, review the instructions in " Converting Projects From WebObjects 4 ," below, to make your applications build under WebObjects 4.5.
