Java Swing

Radar #2824775

CheckBoxMenuItem misbehaves when com.apple.macos.useScreenMenuBar=true.

Description:

When the Menubar has been shifted to the main screen with the com.apple.macos.useScreenMenuBar system property, JCheckBoxMenuItem is triggered twice with the result that selecting the check box turns it on for a moment, and then off again.

Workaround:

This can be worked around by using JRadioButtonMenuItems that have been created with the JcheckBoxMenuItem checkmark, as set in the UIManager. It is important to re-set the global icon for JRadioButtonMenuItems when you are done creating your checkbox "impersonator" so that you can continue to use normal JRadioButtonMenuItems. For Example:

Object radioIcon = UIManager.get("RadioButtonMenuItem.checkIcon");
JRadioButtonMenuItem radio1, radio2, radio3;
UIManager.put("RadioButtonMenuItem.checkIcon", UIManager.get("CheckBoxMenuItem.checkIcon"));
JRadioButtonMenuItem radio2 = new JRadioButtonMenuItem("Check Icon");
UIManager.put("RadioButtonMenuItem.checkIcon", radioIcon);

This information is also available in Technote Q&A number QA1154 available at http://developer.apple.com/qa/indexes/java-a.html.


Radar #2842030

Can't modify submenus in a ScreenMenuBar.

Description:

When using a ScreenMenuBar top level menus are modifiable, but sub menus are not.

Workaround:

Rebuild the entire submenu every time you change it:

Component[] subMenuItems = subMenu.getMenuComponents();
topMenu.remove(subMenu);
subMenu = new JMenu("Open Recent");
for (int i=0; i<subMenuItems.length; i++) {
            subMenu.add(subMenuItems[i]);
}
topMenu.add(subMenu);

More information is available in Technote Q&A number QA1171 available at http://developer.apple.com/qa/indexes/java-a.html.


Radar #2881111

Crashes in Swing applications that use IFrames.

Description:

Swing applications that use IFrame may leak and subsequently crash when using the Metal look and feel. Usually the crash occurs after multiple IFrames have been created.

Resolution:

This has been fixed.


Radar #2883050

Applications may throw an exception when system date is set to a date in the past.

Description:

Applications that use Swing or AWT components may throw an exception if the date is set to a date in the past from System Preferences.

Resolution:

This has been fixed.


Radar #2884768

JMenu.getLocationOnScreen reports incorrect value for Aqua Swing screen menu bar.

Description:

When the menu bar is set to be at the top of the screen, JMenu components in the menu bar may report incorrect coordinates for getLocationOnScreen.

Workaround:

None.
