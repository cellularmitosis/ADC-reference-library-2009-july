ObserverProxy

Extends:
next.eo.DelayedObserver

Class Description

ObserverProxy provides a means for objects that can't inherit from DelayedObserver to handle subjectChanged messages. An ObserverProxy overrides subjectChanged to send an action message to its target object, allowing the target to act as though it had received subjectChanged directly from an DelayedObserverQueue. See the ObserverCenter and DelayedObserverQueue class specifications for more information.

Constructors

ObserverProxy

public next.eo.ObserverProxy(java.lang.Object anObject, next.util.Selector anAction, int priority)

Creates a new ObserverProxy to send anAction to anObject upon receiving a subjectChanged message. anAction should be a selector for a typical action method, taking one java.util.Object argument and returning void. priority indicates when the receiver is sent this message from DelayedObserverQueue's notifyObserversUpToPriority method.
