Display Groups

A display group is a variable that accesses records in a database. It can fetch, insert, delete, display, and update records.

Generally, you'll create a display group when you create your WebObjects project by using a Wizard in Project Builder. However you can also add a display group to your application within WebObjects Builder.


Adding a Display Group

To add a display group that's not in your project:

  • Drag an entity, relationship, or fetch specification from the EOModeler application into your component window.
    Dialogs appear that let you name and configure the display group. For information on configuring, see Configuring the Display Group

To add a display group whose model file is already in your project:

  1. Choose Add Key from the Edit Source pop-up menu, and define a variable of type WODisplayGroup
  2. Configure the display group.

    See Configuring the Display Group.

Configuring the Display Group

Before you can use a display group in your component, you must configure it.

In the object browser, a display group with a check mark beside it has been configured, and a display group with a dash beside it has not. A configured display group shows its keys and actions in the object browser and lets you bind them to elements within your program.

To configure a display group

  • Double-click its name and edit the Display Group Options dialog.

To choose the entity to display, use the Entity field.

To specify that this display group is the detail for a master-detail set, choose "Has detail data source".

To choose the number of records to display on a single page, use the "Entries per batch" field. If it's zero, the display group shows all the records it finds.

To determine how to search when performing a query, use the "Qualification" pop-up menu. You can choose to display records that begin with, end with, or contain the string specified.

To determine when to fetch the objects, use the "Fetches on load" checkbox. If it's selected, the group fetches all its objects as the component is loaded. Note that if your component needs to gather some information to qualify the search, you must deselect this checkbox and perform the search programmatically.

To specify how to sort the records, use the "Sorting" options. Choose the attribute to sort by from the Sorting pop-up menu, and choose the order of sorting with the radio buttons below the menu.

To fetch the records using a fetch specification from the EOModel file, use the "Fetch Spec" pop-up menu.


Using Master and Detail Display Groups

You can have a display group that shows detailed information about an item from another display group. The master displays a group of records from the database. The detail displays a group of records related to a selected item in the master display group by a one-to-many relationship. The detail display group doesn't fetch its data from the database, but gets its data directly from the selected item in the master display group.

For example, the master display group lists movies, and the detail display group lists the actors in a movie. Or the master display group lists a company's departments, and the detail display group lists the employees in a department.

To specify that a display group is a detail display group:

  • In the Display Group Options dialog, use the Master/Detail section.

    Select "Has detail data source," choose the related master entity from the Master Entity pop-up menu, and choose the key that contains the data from the Data Key menu.

You can also create a detail display group by dragging a to-many relationship from EOModeler into your component.
