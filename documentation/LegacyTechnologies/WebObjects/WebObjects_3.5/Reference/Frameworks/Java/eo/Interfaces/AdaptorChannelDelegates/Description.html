AdaptorChannelDelegates

Interface Description

The AdaptorChannelDelegates interface defines methods that an AdaptorChannel can invoke in its delegate. Delegates are not required to provide implementations for all of the methods in the interface. When you write a delegate, you don't have to use the implements keyword to specify that the object implements the AdaptorChannelDelegates interface. Instead, simply use the AdaptorChannel setDelegate method to assign your object as the delegate and then declare and implement any subset of the methods declared in AdaptorChannelDelegates. The AdaptorChannel can determine if the delegate doesn't implement a delegate method and only attempts to invoke the methods the delegate actually implements.

Many of the AdaptorChannel methods notify an adaptor channel's delegate before and after the channel performs an operation. Some delegate methods, such as adaptorChannelShouldEvaluateExpression, let the delegate determine whether the channel should perform an operation. Others, such as adaptorChannelDidEvaluateExpression, are simply notifications that an operation has occurred. The delegate has an opportunity to respond by implementing the delegate methods. If the delegate wants to intervene, it implements adaptorChannelShouldEvaluateExpression. If it simply wants notification when a transaction has begun, it implements adaptorChannelDidEvaluateExpression.

Instance Methods

adaptorChannelDidChangeResultSet

public void adaptorChannelDidChangeResultSet(java.lang.Object channel)

Invoked from channel's fetchRow method when a select operation results in multiple result sets. This method tells the delegate that the next invocation of fetchRow will fetch from the next result set. This method is invoked when fetchRow returns null and there are still result sets left to fetch. The delegate can invoke channel's setAttributesToFetch to prepare the AdaptorChannel for fetching the new rows.

See also: adaptorChannelDidFinishFetching

adaptorChannelDidEvaluateExpression

public void adaptorChannelDidEvaluateExpression(java.lang.Object channel, next.eo.SQLExpression expression)

Invoked from channel's evaluateExpression method to tell the delegate that expression was evaluated by the database server.

See also: adaptorChannelShouldEvaluateExpression

adaptorChannelDidExecuteStoredProcedure

public void adaptorChannelDidExecuteStoredProcedure(java.lang.Object channel, next.eo.StoredProcedure procedure, next.util.ImmutableHashtable values)

Invoked from channel's executeStoredProcedure method after procedure is executed successfully with values, a dictionary of argument names and values.

See also: adaptorChannelShouldExecuteStoredProcedure

adaptorChannelDidFetchRow

public void adaptorChannelDidFetchRow(java.lang.Object channel, next.util.MutableHashtable row)

Invoked from channel's fetchRow method after row is successfully fetched from the database. This method is not invoked if an exception occurs during the fetch or if fetchRow returns null because there are no more rows in the current result set. The delegate may modify row, which will be returned from fetchRow.

See also: adaptorChannelWillFetchRow

adaptorChannelDidFinishFetching

public void adaptorChannelDidFinishFetching(java.lang.Object channel)

Invoked from channel's fetchRow method to tell the delegate that fetching is finished for the current select operation (from any of selectAttributesWithFetchSpecification, executeStoredProcedure, or evaluateExpression). This method is invoked when a fetch ends in fetchRow because there are no more result sets.

adaptorChannelDidPerformOperations

public java.lang.Throwable adaptorChannelDidPerformOperations(java.lang.Object channel, next.util.ImmutableVector operations, java.lang.Throwable exception)

Invoked from channel's performAdaptorOperations method. exception is null if no exception was thrown while operations were performed. Otherwise, exception is the thrown exception. The delegate can return the same or a different exception, which is re-thrown by performAdaptorOperations, or it can return null to prevent the adaptor channel from throwing an exception.

See also: adaptorChannelWillPerformOperations

adaptorChannelDidSelectAttributes

public void adaptorChannelDidSelectAttributes(java.lang.Object channel, next.util.ImmutableVector attributes, next.eo.FetchSpecification fetchSpecification, boolean flag, next.eo.Entity entity)

Invoked from channel's selectAttributesWithFetchSpecification method to tell the delegate that rows have been selected in the database server for fetchSpecification. If flag is true, the rows have been locked in the database.

adaptorChannelShouldConstructStoredProcedureReturnValues

public abstract next.util.ImmutableHashtable adaptorChannelShouldConstructStoredProcedureReturnValues(java.lang.Object channel)

Invoked from channel's returnValuesForLastStoredProcedureInvocation method to tell the delegate that channel is constructing return values for the last stored procedure evaluated. If the delegate returns a value other than null, that value will be returned immediately from returnValuesForLastStoredProcedureInvocation.

See also: adaptorChannelShouldReturnValuesForStoredProcedure

adaptorChannelShouldEvaluateExpression

public boolean adaptorChannelShouldEvaluateExpression(java.lang.Object channel, next.eo.SQLExpression expression)

Invoked from channel's evaluateExpression method to tell the delegate that channel is sending an expression to the database server. The delegate returns true to permit the adaptor channel to send expression to the server. If the delegate returns false, the adaptor channel does not send the expression and returns immediately. When the delegate returns false, the adaptor channel expects that the delegate has done the work that evaluateExpression would have done. The delegate can create a new SQLExpression and send the expression itself before returning false.

See also: adaptorChannelDidEvaluateExpression

adaptorChannelShouldExecuteStoredProcedure

public abstract next.util.ImmutableHashtable adaptorChannelShouldExecuteStoredProcedure(java.lang.Object channel, next.eo.StoredProcedure procedure, next.util.ImmutableHashtable values)

Invoked from channel's executeStoredProcedure method to tell the delegate that channel is executing a stored procedure. If the delegate returns a value other than null, that value is used as the arguments to the stored procedure instead of values.

See also: adaptorChannelDidExecuteStoredProcedure

adaptorChannelShouldReturnValuesForStoredProcedure

public next.util.ImmutableHashtable adaptorChannelShouldReturnValuesForStoredProcedure(java.lang.Object channel, next.util.ImmutableHashtable returnValues)

Invoked from channel's returnValuesForLastStoredProcedureInvocation method to tell the delegate that channel is returning values for a stored procedure. If the delegate returns a value other than null, that value is returned from returnValuesForLastStoredProcedureInvocation instead of returnValues.

See also: adaptorChannelShouldConstructStoredProcedureReturnValues

adaptorChannelShouldSelectAttributes

public abstract boolean adaptorChannelShouldSelectAttributes(java.lang.Object channel, next.util.ImmutableVector attributes, next.eo.FetchSpecification fetchSpecification, boolean flag, next.eo.Entity entity)

Invoked from channel's selectAttributesWithFetchSpecification method to ask the delegate whether a select operation should be performed. The delegate should not modify fetchSpecification. Instead, if the delegate wants to perform a different select it should invoke selectAttributesWithFetchSpecification itself with a new fetch specification, and return false (indicating that the adaptor channel should not perform the select itself).

See also: adaptorChannelDidSelectAttributes

adaptorChannelWillFetchRow

public void adaptorChannelWillFetchRow(java.lang.Object channel)

Invoked from channel's fetchRow method to tell the delegate that a single row will be fetched. The delegate can determine the attributes used by the fetch by sending attributesToFetch to channel, and can change the set of attributes to fetch by sending setAttributesToFetch to channel. The adaptor channel performs the actual fetch.

See also: adaptorChannelDidFetchRow

adaptorChannelWillPerformOperations

public next.util.ImmutableVector adaptorChannelWillPerformOperations(java.lang.Object channel, next.util.ImmutableVector operations)

Invoked from channel's performAdaptorOperations method to tell the delegate that channel is performing the AdaptorOperations in operations. The delegate may return operations or a different array of operations for the adaptor channel to perform. If the delegate returns null, the adaptor channel does not perform the operations and returns from the method immediately.

See also: adaptorChannelDidPerformOperations
