Images

WebObjects Builder lets you create three types of images:

  • A static image (<IMG>) refers to a known pathname
  • A dynamic image (WOImage) refers to an image in your project or a framework. If the image's name will be known only when WebObjects creates the page, supply the name with a key.
  • An dynamic active image (WOActiveImage) is like a dynamic image, but also has an image map.

To create a static image:

  • Choose Elements > Image or click the Static Image button, and select the image.

In the static image's Inspector, you can change the image's size and file path and whether it uses an absolute or relative reference.

To add a dynamic image whose file is already in your project:

  • Choose WebObjects > WOImage or click the Dynamic Image button.
  • Set the image's filename attribute.

    If the filename is known now, bind filename to it.

    If the filename will be known when WebObjects generates the page, bind filename to a key that contains the name.

To add a dynamic image whose file is not in your project:

  • Drag the image from the Finder to your component page.

The map file for a dynamic active image must be in the same directory or framework as the image file. By default, the map file has the same name as the image file with the extension .map. If the map file has a different name, bind the attribute imageMapFile to it.

To set an image for the page background, see Pages.
