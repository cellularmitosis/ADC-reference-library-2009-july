Class Description

KeyValueQualifier is a subclass of Qualifier that compares a named property of an object with a supplied value, for example, "salary > 1500". KeyValueQualifier implements the QualifierEvaluation interface, which defines the method evaluateWithObject for in-memory evaluation. When a KeyValueQualifier object receives an evaluateWithObject message, it evaluates the given object to determine if it satisfies the qualifier criteria.

In addition to performing in-memory filtering, KeyValueQualifier can be used to generate SQL. When it's used for this purpose, the key should be a valid property name of the root entity for the qualifier (or a valid key path).

KeyValueQualifier

public next.eo.KeyValueQualifier(java.lang.String key, next.util.Selector selector, java.lang.Object value)

public next.eo.KeyValueQualifier(next.util.Coder aCoder)

Creates and returns a new KeyValueQualifier.

If key, selector, and value are provided, the KeyValueQualifier compares values for key to value using the operator method selector. The possible values for selector are as follows:

  • QualifierOperatorEqual
  • QualifierOperatorNotEqual
  • QualifierOperatorLessThan
  • QualifierOperatorGreaterThan
  • QualifierOperatorLessThanOrEqualTo
  • QualifierOperatorGreaterThanOrEqualTo
  • QualifierOperatorLike
Enterprise Objects Framework supports SQL generation for these methods only. You can generate SQL using the SQLExpression static method sqlStringForKeyValueQualifier.

For example, the following excerpt creates a KeyValueQualifier qual that has the key "name", the operator method QualifierOperatorEqual, and the value "Smith". Once constructed, the qualifier qual is used to filter an in-memory array.

ImmutableVector employees /* Assume this exists */
KeyValueQualifier qual = new KeyValueQualifier("name", 
    Qualifier.QualifierOperatorEqual, "Smith");
return Qualifier.filteredVectorWithQualifier(employees, qual);

If aCoder is provided, the new KeyValueQualifier is decoded from aCoder. This constructor, together with encodeWithCoder, provide the basis for archiving and distributing qualifiers. aCoder is specified when a KeyValueQualifier is being read from an archive file; you never specify this constructor directly. For more information, see the next.util.Coding interface specification in the Foundation Framework Reference.

evaluateWithObject

QualifierEvaluation interface

public boolean evaluateWithObject(java.lang.Object anObject)

Returns true if the object anObject satisfies the qualifier, false otherwise. When a KeyValueQualifier object receives the evaluateWithObject message, it evaluates anObject to determine if it meets the qualifier criteria. This method can throw one of several possible exceptions if an error occurs. If your application allows users to construct arbitrary qualifiers (such as through a user interface), you may want to write code to catch any exceptions and properly respond to errors (for example, by displaying a panel saying that the user typed a poorly formed qualifier).
