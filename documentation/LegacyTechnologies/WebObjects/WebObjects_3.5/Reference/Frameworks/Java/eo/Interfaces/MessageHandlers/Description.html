MessageHandlers

Interface Description

The MessageHandlers interface declares methods used for error reporting and determining fetch limits. See the EditingContext, DatabaseContext, and EODisplayGroup (in the Objective-C version of the Enterprise Objects Framework Reference) class specifications for more information.

Instance Methods

Message handlers are primarily used to implement exception handling in the interface layer's , and wouldn't ordinarily be used outside the scope of an OpenStep application (not in a command line tool or WebObjects application, for example).

editingContextShouldContinueFetching

public abstract boolean editingContextShouldContinueFetching(next.eo.EditingContext anEditingContext, int count, int limit, next.eo.ObjectStore objectStore)

Invoked by an objectStore (such as a DatabaseContext) to allow the message handler for anEditingContext (often an EODisplayGroup) to prompt the user about whether or not to continue fetching the current result set. The count argument is the number of objects fetched so far. limit is the original limit specified using the FetchLimitHintKey in the hint dictionary of the FetchSpecification (see the DatabaseContext class specification for more information).

presentErrorMessageForEditingContext

public abstract void presentErrorMessageForEditingContext(next.eo.EditingContext anEditingContext, java.lang.String message)

Invoked by anEditingContext, the implementation of this method should present message to the user in whatever way appropriate (whether by opening an attention panel or printing the message in a terminal window, for example).
