Java Events

Radar #2778552

Modal dialogs in drop methods may cause a freeze.

Description:

Displaying a modal dialog in the drop method of a DragTargetListener causes the Java application to freeze. It also stops the Dock from being able to launch or switch applications.

Workaround:

Move the dialog to a separate thread.


Radar #2832782

No click thru with Java on Mac OS X.

Description:

On other Java platforms you have a click-thru behavior. This is especially useful for palette windows.

Workaround:

Currently a user must click twice in Java applications, once to obtain focus and then a second time to initiate the desired action.


Radar #2908518

Event modifier behavior has changed.

Description:

The behavior of the event modifiers has changed. For example, AWT InputEvent isPopupTrigger method reports true for option and ctrl-clicks return true for isAltDown().

Workaround:

If isPopupTrigger() checks are preempting your need to detect option (in a MouseListener, DragGestureRecognizer, etc.) make sure you check explicitly for option/ALT before checking isPopupTrigger(). Since ctrl also registers isAltDown() as true, you need to do the following:

public void mousePressed(MouseEvent e) {
    // Option key ONLY
    // workaround for Radar 2908518
    if (e.isAltDown() && !e.isCtrlDown()) {
        // nop to let DnD activate, or special option-click handling
    } else if (e.isPopupTrigger()) {
        // context menu AFTER confirming this is not an option-click
    }
}
