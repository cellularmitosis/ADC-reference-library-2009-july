CLASS DESCRIPTION

A next.wo.DisplayGroup is the basic user interface manager for a WebObjects application that accesses a database. It collects objects from a next.eo.DataSource, filters and sorts them, and maintains a selection in the filtered subset. You bind WebObjects dynamic elements to DisplayGroup attributes and methods to display information from the database on your web page.

A DisplayGroup manipulates its DataSource by sending it fetchObjects, insertObject, and other messages, and registers itself as an editor and message handler of the DataSource's next.eo.EditingContext. The EditingContext then monitors the DisplayGroup for changes to objects.

Most of a DisplayGroup's interactions are with its DataSource and its EditingContext. See the next.eo.DataSource, and next.eo.EditingContext class specifications for more information on these interactions.

Creating a DisplayGroup

You create most DisplayGroups in WebObjects Builder in one of these ways:

  • By using the Database Wizard to create the DisplayGroup and the dynamic elements that are bound to it.

  • By dragging an entity icon from the EOModeler application to create a DisplayGroup associated with the model's data source.

  • By choosing the Tools->Script->Add Display Group menu item to create a DisplayGroup not bound to a data source.

DisplayGroups with DataSources operate independent of other DisplayGroups, while those without DataSources must be set up in a master-detail association with another DisplayGroup. For more information on setting up DisplayGroups in WebObjects Builder, see:

  • "Creating a Web-Based Database Application With WebObjects Builder" in Getting Started.

  • "Writing WebScript in WebObjects Builder" in Using WebObjects Builder.

To create a DisplayGroup programmatically, simply initialize it and set its DataSource: 

import next.eo.*;
import next.wo.*;

...

DataSource myDataSource;    /* Assume this exists. */
DisplayGroup myDisplayGroup;

myDisplayGroup = DisplayGroup();
myDisplayGroup.setDataSource(myDataSource);

Getting Objects

Since a DisplayGroup isn't much use without objects to manage, the first thing you do with a DisplayGroup is send it a fetch message. You can use the basic fetch method, or you can configure the DisplayGroup in WebObjects Builder to fetch automatically when its component is loaded. The fetch method asks the DisplayGroup's DataSource to fetch from its persistent store with a fetchObjects message.

Filtering and Sorting

A DisplayGroup's fetched objects are available through its allObjects method. These objects are treated only as candidates for display, however. The vector of objects actually displayed is filtered and sorted by the DisplayGroup's delegate or by a qualifier and a sort ordering vector. You set the qualifier and sort orderings using the setQualifier and setSortOrdering methods. The displayedObjects method returns this filtered and sorted vector; index arguments to other DisplayGroup methods are defined in terms of this vector.

If the DisplayGroup has a delegate that responds to displayArrayForObjects (defined in the next.wo.DisplayGroupDelegate interface), it invokes this method rather than using its own qualifier and sort ordering vector. The delegate is then responsible for filtering the objects and returning a sorted vector. If the delegate only needs to perform one of these steps, it can get the qualifier or sort orderings from the DisplayGroup and apply either itself using the next.eo.Qualifier method vectorByFilteringWithQualifier and the next.eo.SortOrdering method sortedVectorUsingKeyOrders.

If you change the qualifier or sort ordering, or alter the delegate in a way that changes how it filters and sorts the DisplayGroup's objects, you can send updateDisplayedObjects to the DisplayGroup to get it to refilter and resort its objects. Note that this doesn't cause the DisplayGroup to refetch.

Batching the Output

Even after the output has been qualifier and sorted, there might be too many objects to display on a single HTML page. For this reason and to improve your application's performance, DisplayGroup supports batching the display. When batching is used, the HTML page shows only a certain number of objects at a time. If you create the DisplayGroup using WebObjects Builder, the default batch size is 10. Thus if there are one hundred objects to display, displayedObjects returns only the first ten of those objects when the page is drawn. If displayNextBatch is invoked, displayedObjects is updated to contain the next ten objects, and the page is redrawn.

You use displayNextBatch and displayPreviousBatch to move back and forth through the displayed objects. The setNumberOfObjectsPerBatch method changes the batch size. You can also change batch size using WebObjects Builder.

Changing and Examining the Selection

A DisplayGroup keeps a selection in terms of indexes into the vector of displayed objects. Components that display values for multiple objects are responsible for updating the selection in their DisplayGroups according to user actions on their dynamic elements. This is typically done with the setSelectionIndexes method. Other methods available for indirect manipulation of the selection are the action methods selectNext and selectPrevious.

To get the selection, you can use the selectionIndexes method, which returns a vector of Integers, or selectedObjects, which returns a vector containing the selected objects themselves. Another method, selectedObject, returns the first selected object if there is one.

Note: The selection is a programmatical selection only. The selected objects don't actually appear highlighted on the web page.

The Delegate

The interface next.wo.DisplayGroupDelegate offers a number of methods for the DisplayGroup's delegate to implement; if the delegate does, DisplayGroup invokes them as appropriate. Besides the aforementioned displayArrayForObjects, there are methods that inform the delegate that the DisplayGroup has fetched, created an object (or failed to create one), inserted or deleted an object, changed the selection, or set a value for a property. There are also methods that request permission from the delegate to perform most of these same actions. The delegate can return true to permit the action or false to deny it. See each method's description in the next.wo.DisplayGroupDelegate interface specification for more information.

Performing Queries

If you are building a component that allows users to perform database queries, you use a DisplayGroup to construct a qualifier that reflects the query.

Instead of performing the fetch method and then using updateDisplayedObjects to qualify and sort the objects returned by fetch, you use executeQuery. The executeQuery method builds a qualifier based on user input, performs a fetch using that qualifier, and then invokes the setQualifier method to set the DisplayGroup's qualifier to null. In this way, subsequent updateDisplayedObjects messages apply only the sort ordering to the displayed objects vector, as the qualifier has already been applied.

The method inputObjectForQualifier specifies the input values used to build the query. inputObjectForQualifier returns a MutableHashTable that you can modify to specify the query. Typically, you modify the hashtable through WebObjects Builder, by binding the hashtable's possible keys to dynamic elements in the component. When the user enters or chooses a value from the dynamic element, inputObjectForQualifier's hashtable is updated to include the appropriate key/value pair.

If you want to support "from-to" queries, the hashtable returned by inputObjectForQualifier specifies only the values in the "from" part of the query. To specify the values for the "to" part of the query, use the hashtable returned by the secondObjectForQualifier method. You bind objects to the hashtable returned by secondObjectForQualifier in the same way that you bind object to inputObjectForQualifier. Because you are specifying a "from-to" query, the keys in the secondObjectForQualifier hashtable must be identical to the keys in the inputObjectForQualifier.

For example, you might use secondObjectForQualifier if you were trying to construct a query that returned all movies that were released between two dates. Both the inputObjectForQualifier and the secondObjectForQualifier hashtables would have the "dateReleased" property. The inputObjectForQualifier value would be the starting date, and the secondObjectForQualifier value would be the ending date.

The qualifier built by executeQuery differs from the qualifier associated with the DisplayGroup (using setQualifier) in when and where it is applied. The qualifier associated with the DisplayGroup filters objects after they are fetched into memory. The qualifier built by executeQuery is associated with the DataSource and thus is applied before the objects are fetched into memory. You can use both qualifiers on the same DisplayGroup; however, after you perform an executeQuery the qualifier associated with the DisplayGroup is set to null so that subsequent updateDisplayedObjects messages apply only the sort ordering to update the displayed objects vector.

Returns the index of the batch currently being displayed. The total batch count equals the number of displayed objects divided by the batch size. For example, if the DisplayGroup has one hundred objects to display and the batch size is twenty, there are five batches. The first batch has a batch index of 1.

See Also: batchCount, numberOfObjectsPerBatch, setCurrentBatchIndex

Attempts to delete the object at index, returning true if successful and false if not. Checks with the delegate using shouldDeleteObject (defined in the next.wo.DisplayGroupDelegate interface). If the delegate returns false, this method fails and returns false. If successful, sends the delegate a didDeleteObject message.

This method performs the delete by sending deleteObject to the next.eo.DataSource. If that message raises an exception, this method fails and returns false.

Increments the current batch index, displays that batch of objects, and clears the selection. If batch currently being displayed is the last batch, this method displays the first batch of objects. Returns null to force the page to be reloaded.

See Also: batchCount, currentBatchIndex, displayBatchContainingSelectedObject, displayPreviousBatch

public java.lang.Object displayPreviousBatch()

displayedObjects

public next.util.ImmutableVector displayedObjects()

Returns the objects that should be displayed or otherwise made available to the user, as filtered by the receiver's delegate, by the receivers qualifier and sort ordering, or by a query built using inputObjectForQualifier and secondObjectForQualifier.

If batching is in effect, displayedObjects returns the current batch of objects.

See Also: allObjects, updateDisplayedObjects, qualifier, qualifierFromInputValues, sortOrdering, displayArrayForObjects (in next.wo.DisplayGroupDelegate interface)

endEditing

public boolean endEditing()

Ends any editing taking place and returns true.

executeQuery

public java.lang.Object executeQuery()

Builds a qualifier using qualifierFromInputValues and then fetches the objects that match that qualifier. The DisplayGroup's qualifier is set to null at the end of this method so that subsequent updateDisplayedObjects messages only apply the sort ordering. Returns null to force the page to be reloaded.

qualifierFromInputValues builds a qualifier using inputObjectForQualifier and secondObjectForQualifier. It does not use the DisplayGroup's qualifier (the one returned by the qualifier method).

See Also: buildsQualifierFromInput, fetch, lastQualifierFromInputValues, setBuildsQualifierFromInput

fetch

public java.lang.Object fetch()

Attempts to fetch objects from the next.eo.DataSource.

Before fetching, invokes endEditing and sends displayGroupShouldFetch (defined in the next.wo.DisplayGroupDelegate interface) to the delegate. If both of those methods were successful, it then sends a fetchObjects message to the receiver's next.eo.DataSource to replace the object vector, and if successful sends the delegate a didFetchObjects message.

This method returns null to force the page to be reloaded.

See Also: allObjects, executeQuery, updateDisplayedObjects

fetchesOnLoad

public boolean fetchesOnLoad()

Returns true if the receiver fetches automatically after being loaded, false if it must be told explicitly to fetch. The default is true. You can set this behavior in WebObjects Builder using the Display Group Options panel.

See Also: fetch, setFetchesOnLoad

inputObjectForQualifier

public next.util.MutableHashtable inputObjectForQualifier()

Returns a MutableHashTable that is used by qualifierFromInputValues to build a qualifier. The MutableHashTable contains keys and values that the user has entered as a query. For example, if the user wants a list of all movies released in 1995 that made over $100 million, the hashtable returned by inputObjectForQualifier would contain two key/value pairs: releaseDate = "1995" and gross = "$100,000,000.00".

When the DisplayGroup receives an executeQuery message, it uses qualifierFromInputValues to build a query from the key/value pairs in the hashtable returned by this method.

To construct a "from-to" query, use the hashtable returned by this method to construct the "from" part of the query and the hashtable returned by secondObjectForQualifier to construct the "to" part of the query.

See Also: buildsQualifierFromInput, lastQualifierFromInputValues, setBuildsQualifierFromInput

insert

public java.lang.Object insert()

Asks the receiver's next.eo.DataSource to create a new object by sending it a createObject message, then inserts the new object just past the first index in the selection, or at 0 if there's no selection. If a new object can't be created, this method sends the delegate a createObjectFailedForDataSource message (defined in the next.wo.DisplayGroupDelegate interface).

If the object is successfully created, this method then sets the default values specified by insertedObjectDefaultValues.

This method returns null to force the page to be reloaded.

insertedObjectDefaultValues

public next.util.ImmutableHashtable insertedObjectDefaultValues()

Returns the default values to be used for newly inserted objects. If the hashtable returned by this method is empty, the insert method adds an object that is initially empty. Because the object is empty, the display group has no value to display on the HTML page for that object, meaning that there is nothing for the user to select and modify. Use the setInsertedObjectDefaultValues method to set up a default value so that there is something to display on the page.

lastQualifierFromInputValues

public next.eo.Qualifier lastQualifierFromInputValues()

If qualifierFromInputValues has previously been invoked to construct a qualifier, this method returns the qualifier that that method built. If qualifierFromInputValues has not been used, this method return null.

localKeys

public next.util.ImmutableVector localKeys()

Returns the additional keys that dynamic elements can be bound to. A DisplayGroup's basic keys are typically those of the attributes and relationships of its objects, as defined by their class descriptions through an entity in the model. Local keys are typically used to form associations with key paths, with arbitrary methods of objects, or with properties of objects not associated with an entity.

Hint: You can use this qualifier after the executeQuery method to further qualify the displayed objects. Use setQualifier to set the qualifier and then use updateDisplayedObjects to apply the qualifier and sort ordering. If you want to further restrict the displayed objects, you can join this qualifier with qualifier from previous queries.

See Also: setLocalKeys

secondObjectForQualifier

public next.util.MutableHashtable secondObjectForQualifier()

Returns a MutableHashTable that specifies the "to" value in from-to queries. For example, suppose the user has built a query to return all movies released between two dates: January 1, 1994 and December 31, 1994. The hashtable returned by secondObjectForQualifier would contain one key (such as "dateReleased") and the value for the key would be December 31, 1994. The hashtable returned by this method must always have the exactly the same keys as the hashtable returned by the inputObjectForQualifier method.

This method is used by the qualifierFromInputValues method to build a qualifier.

See Also: buildsQualifierFromInput, executeQuery, setBuildsQualifierFromInput

selectNext

public java.lang.Object selectNext()

Attempts to select the object just after the currently selected one. The selection is altered in this way:

  • If there are no objects, does nothing.

  • If there's no selection, selects the object at index zero.

  • If the first selected object is the last object in the displayed objects vector, selects the first object.

  • Otherwise selects the object after the first selected object.

This method returns null to force the page to be reloaded.

See Also: selectPrevious, setSelectionIndexes

selectObject

public boolean selectObject(java.lang.Object anObject)

Attempts to select the object equal to anObject in the receiver's displayed objects vector, returning true if successful and false otherwise. anObject is equal to an object in the displayed objects vector if its address is the same as the object in the vector.

See Also: selectNext, selectPrevious

selectPrevious

public java.lang.Object selectPrevious()

Attempts to select the object just before the presently selected one. The selection is altered in this way:

  • If there are no objects, does nothing.

  • If there's no selection, selects the object at index zero.

  • If the first selected object is at index zero, selects the last object.

  • Otherwise selects the object before the first selected object.

This method returns null to force the page to be reloaded.

See Also: selectNext, redisplay

selectedObject

Returns the first selected object in the displayed objects vector, or null if there's no such object.

See Also: displayedObjects, selectionIndexes, selectedObjects

setObjectArray

public void setObjectArray(next.util.ImmutableVector objects)

Sets the receiver's objects to objects, regardless of what its next.eo.DataSource provides. This method doesn't affect the DataSource's objects at all; specifically, it results in neither inserts nor deletes of objects in the DataSource. objects should contain objects with the same property names or methods as those accessed by the receiver. This method is used by fetch to set the vector of fetched objects; you should rarely need to invoke it directly.

After setting the object vector, this method restores as much of the original selection as possible. If there's no match and the receiver selects after fetching, then the first object is selected.

public void setQualifier(next.eo.Qualifier aQualifier)

Sets the receiver's qualifier to aQualifier. This qualifier is used to filter the receiver's vector of objects for display. Use updateDisplayedObjects to apply the qualifier.

If the receiver's delegate responds to displayArrayForObjects (defined in the next.wo.DisplayGroupDelegate interface), that method is used instead of the qualifier to filter the objects.

If you use executeQuery to perform a query based on use input, the qualifier set by this method is discarded.

See Also: displayedObjects, qualifier

setSelectionIndexes

public boolean setSelectionIndexes(next.util.ImmutableVector indexes)

Selects the objects at indexes in the receiver's vector if possible, returning true if successful and false if not (in which case the selection remains unaltered). indexes is a vector of java.lang.Integers. This method is the primitive method for altering the selection; all other such methods invoke this one to make the change.

This method invokes endEditing to wrap up any changes being made by the user. If endEditing returns false, this method fails and returns false. This method then checks the delegate with a shouldChangeSelectionToIndexes message (defined in the next.wo.DisplayGroupDelegate interface). If the delegate returns false, this method also fails and returns false. If the receiver successfully changes the selection, its observers each receive a subjectChanged message.

Note: The selection set here is only a programmatic selection; the objects on the screen are not highlighted in any way.

See Also: allObjects
