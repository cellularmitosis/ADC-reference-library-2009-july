Java Toolkit

Radar #2510039

MRJFileUtils should include getCreationDate and setCreationDate methods.

Description:

The com.apple.mrj package does not contain methods to set and get the creation date of files.

Workaround:

You can use either JDirect or Carbon to set and get the creation date of files.


Radar #2585075

MRJFileUtils.setDefaultFileType, setDefaultFileCreator don't work in OS X.

Description:

Certain functions present in Mac OS 9's Java implementation no longer work in Mac OS X.

Workaround:

In Mac OS X, application behavior should not be dictated by a file type code only. These methods depend on a certain filesytem, HFS(+). Since Mac OS X can run on other filesystems, these methods should not be used.


Radar #2868805

MRJQuitHandler handleQuit() method causes hang.

Description:

Implementing the MRJQuitHandler interface can lead to an application hanging (spinning color wheel cursor) under specific circumstances when quitting the application.

Workaround:

This has been fixed.


Radar #2869229

MRJQuitHandler may create a default application instance.

Description:

It's been reported that warnings about default application being created have been received in applications using MRJQuitHandler.

Workaround:

This has been fixed.
