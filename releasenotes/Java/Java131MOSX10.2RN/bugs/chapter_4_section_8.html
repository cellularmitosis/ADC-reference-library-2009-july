Java Frames

Radar #2869281

Java 1.3.1 Update 1 changes JDialog behavior.

Description:

Problems with JDialog have been reported in Java Swing applications after the Java 1.3.1 update was installed.

Workaround:

This has been fixed.


Radar #2870610

Application hangs when restoring JDialogs from the Dock.

Description:

Various problems with JDialogs have been reported. For example, if you put more than one non-modal JDialog in the Dock and restore them by calling setVisible(true) and toFront(), the application hangs when it tries to restore the second JDialog.

Workaround:

This has been fixed.


Radar #2879484

Application hangs while activating another JDialog.

Description:

Problems have been reported using Java Swing applications that consist of a JFrame and non-modal JDialogs that have JFrame as their owner. Occasionally the application completely hangs when one of the dialogs is activated. Various freezes have also been reported that occur when a component is clicked or selected in applications, for example, that display a JFrame that contains one JButton. When the button is clicked a JDialog appears that contains a JMenuBar. The application hangs after repeatedly clicking the button and a menu of the menu bar.

Workaround:

Can be worked around by adding a WindowListener to your frame. Simply call toFront() on your window in your windowActivated() method to ensure your window actually has focus when it is selected, before any other UI actions are performed. Please note that this is a rare problem and should only be applied to areas that demonstrate the problem, not as a general practice. Technote Q&A number QA1153 available at http://developer.apple.com/qa/indexes/java-a.html discusses this issue.
