Technical Q&A OPS22
Notification Manager Reinitialized During Boot

Q: Starting with Mac OS 7.5, the Notification Manager queue seems to be reinitialized during boot. Is this really what is happening?

A: Yes, that is correct. If you install anything into the Notification Manager’s queue before IAZNotify time (such as a SCSI or ATA driver loading from disk during boot), you will find that before the first INIT is loaded that the Notification Manager’s queue is reinitialized and your entry has been disposed of.

You will have to install your notification at a later time, such as accRun time.

This bug [2343611] was fixed in Mac OS 9.0.
