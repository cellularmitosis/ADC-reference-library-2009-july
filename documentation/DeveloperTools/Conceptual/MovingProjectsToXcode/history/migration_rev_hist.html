This table describes the changes to Porting CodeWarrior Projects to Xcode.

DateNotes
2006-10-26Corrected errors. Changed title from "Moving CodeWarrior Projects to Xcode."
 

Updated “For Best Results When Importing” and “Known Issues With the Importer” to note that CodeWarrior must be running when importing.

2006-11-07Corrected errors.
 

Revised statement of pragma support in GCC.

2006-09-05Corrected errors.
 

Fixed missing semicolon in Listing 4-3.

 

Updated “Known Issues With the Importer” to reflect importer support for CW object targets.

2006-05-23Made document-flow change.
 

Changed the order of the steps to update the PowerPlant source files in “Make Changes to PowerPlant” to make them easier to follow.

2005-11-09Updated for Xcode 2.2 and corrected errors.
 

Added “Using PowerPlant in Universal Binaries.”

 

Updated “Exporting Symbols” to reflect support for importing .exp files in Xcode 2.2.

 

Corrected description in “C and C++ Libraries” of options passed by Xcode to the compiler when changing filetype.

 

In “Working With Resources,” added links to additional information on resource files and build phases.

 

Added note, in “Known Issues With the Importer,” about the importer not supporting targets whose Project Type is Object.

 

Added note about including ppcintrinsics.h to use intrinsics in Xcode.

 

Fixed typos.

2005-08-11Fixed typos and made minor editorial corrections.
 

Updated “Building Code” to reflect default Imported CodeWarrior Settings build configuration for imported projects.

 

Fixed typos.

2005-06-06Updated for Xcode 2.1 and GCC 4.0.
 

Added section on “Build Configurations”; removed mention of build styles.

 

Updated description of Search field in “Some Special Features of Xcode” to include wildcard and regular expression searching.

 

Added descriptions of project window layouts and showing / hiding smart groups to “Customizing the Environment.”

 

Revised discussion of header files and search paths in “Header Files” to reflect Xcode 2.0 support for recursive search paths.

 

Updated build phase names throughout.

 

Added mention of breakpoint actions and watchpoints to “Debugging.”

 

Updated “The GCC Compiler” for GCC 4.0; also added note about removal of GCC 3.1 and 2.95.

 

Noted that prebinding is no longer necessary if you are running on 10.3.4 or later in “Prebinding.”

 

Updated “C and C++ Libraries” to reflect the switch to packaging the standard C++ library as a dynamic shared library.

Updated for July 2004 Xcode Tools.

 

Updated screenshots.

 

Added information on using dead code stripping in Xcode to section “Dead Code Stripping.”

 

Added a note about a Rez bug to section “Working With Resources.”
