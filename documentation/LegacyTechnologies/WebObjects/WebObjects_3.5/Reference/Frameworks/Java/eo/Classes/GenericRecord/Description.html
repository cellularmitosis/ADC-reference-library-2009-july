Class Description

GenericRecord is a generic enterprise object class that can be used in place of custom classes when you don't need custom behavior. It implements the EnterpriseObject interface to provide the basic enterprise object behavior. A GenericRecord object has a ClassDescription that provides metadata about the generic record, including the name of the entity that the generic record represents and the names of the record's attributes and relationships. A generic record stores its properties in a dictionary using its attribute and relationship names as keys.

In the typical case of applications that access a relational database, the access layer's modeling objects are an important part of how generic records map to database rows: If a Model doesn't have a custom enterprise object class defined for a particular entity, a DatabaseChannel using that model creates GenericRecords when fetching objects for that entity from the database server. During this process, the DatabaseChannel also sets each generic record's classDescription to an EntityClassDescription, providing the link to the record's associated modeling objects.

Creating an Instance of GenericRecord

The best way to create an instance of GenericRecord is using the ClassDescription method createInstanceWithEditingContext as follows:

EnterpriseObject newEO;
String entityName;       // Assume this exists.

ClassDescription description = ClassDescription.descriptionForEntityName(entityName)
newEO = description.createInstanceWithEditingContext(null, null);

createInstanceWithEditingContext is preferable to using a GenericRecord constructor because the same code works if you later use a custom enterprise object class instead of GenericRecord. You can get a ClassDescription for an entity name as shown above. Alternatively, you can get a ClassDescription for a destination key of an existing enterprise object as follows:

EnterpriseObject newEO;
EnterpriseObject existingEO;   // Assume this exists.
String relationshipName;       // Assume this exists.
ClassDescription sourceDesc = existingEO.classDescription();
ClassDescription desc = sourceDesc.classDescriptionForDestinationKey(relationshipName);

newEO = desc.createInstanceWithEditingContext(null, null);

The technique in this example is useful for inserting a new destination object into an existing enterprise object-for creating a new Movie object to add to a Studio's array of Movies, for example.

Constructors

GenericRecord

public next.eo.GenericRecord(next.eo.EditingContext anEditingContext, next.eo.ClassDescription aClassDescription, next.eo.GlobalID globalID)

public next.eo.GenericRecord(next.util.Coder aCoder)

Creates a new GenericRecord. If aClassDescription is provided, the new GenericRecord gets its metadata from aClassDescription. You can pass null for anEditingContext and globalID, because the arguments are optional: GenericRecord's implementation does nothing with them. Throws an exception if aClassDescription is null.

If aCoder is provided, the new GenericRecord is decoded from aCoder. This constructor, together with encodeWithCoder, provide the basis for archiving and distributing qualifiers. aCoder is specified when a GenericRecord is being read from an archive file; you never specify this constructor directly. For more information, see the next.util.Coding interface specification in the Foundation Framework Reference.

You shouldn't use these constructors to create new GenericRecords. Rather, use ClassDescription's createInstanceWithEditingContext method. See the class description for more information.

Instance Methods

takeValueForKey

public void takeValueForKey(java.lang.Object value, java.lang.String key)

Invokes the receiver's willChange method, and sets the value for the property identified by key to value. If value is null, this method removes the receiver's dictionary entry for key. If key is not one of the receiver's attribute or relationship names, GenericRecord's implementation does not invoke handleTakeValue. Instead, GenericRecord's implementation does nothing.

See also: valueForKey

valueForKey

public java.lang.Object valueForKey(java.lang.String key)

Returns the value for the property identified by key. If key is not one of the receiver's attribute or relationship names, GenericRecord's implementation does not invoke handleQueryWithUnboundKey. Instead, GenericRecord's implementation simply returns null.
