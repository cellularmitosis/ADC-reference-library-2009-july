Class Description

A TemporaryGlobalID identifies a newly created object before it's saved to an external store. When the object is saved, the temporary ID is converted to a permanent one, as described in the GlobalID class specification.

public next.eo.TemporaryGlobalID(next.util.Coder aCoder)

Creates and returns a TemporaryGlobalID as a unique instance. The returned object contains a byte string that's guaranteed to be unique network-wide. As a result, TemporaryGlobalIDs can be safely passed between processes and machines while still preserving global uniqueness. The returned byte string has the format:

< Sequence [2], ProcessID [2] , Time [4], IP Addr [4] >

If aCoder is provided, the new TemporaryGlobalID is decoded from aCoder. This constructor, together with encodeWithCoder, provide the basis for archiving and distributing qualifiers. aCoder is specified when a TemporaryGlobalID is being read from an archive file; you never specify this constructor directly. For more information, see the next.util.Coding interface specification in the Foundation Framework Reference.
