Managing Keys and Actions

A key is either an instance variable or a method that returns a value. An action is a method that takes no parameters and returns a component (the next page to be displayed).

You can manage them in the object browser at the bottom of the Component window. You can manage your keys and actions by Control-clicking on one in the object browser and using commands from the menu that appears.


Adding a Key

In the object browser, you can add a key to any object. And if you add instance variables to these classes in your source code, the object browser displays them.

To add a key to the component:

  • Choose Add Key from the Edit Source menu at the bottom of the window

To add a key to the application or session:

  • In the object browser, select application or session, Control-click on the second column, and choose Add Key.

When you choose Add Key, a dialog appears that lets you choose the name and type of the key, as well as how you'll access it. Keep these points in mind:

  • If you want classes other than your component class to use the key, create methods to get and set its value with the options "A method returning the value" and "A method setting the value."
  • By default, the method to get the value has the same name as the key, and the method to set the value has the same name as the key with "set" prepended to it. If you want the method returning the key to have "get" prepended to its name, use the "Prepend 'get' to method name" option.
  • If the key is a derived value (that is, its value is derived from keys that already exist), you can deselect "An instance variable," select "A method returning the value," and then edit the source code to return the correct value.

For example, if you create the key myVar and select the options to create an instance variable, a method returning the value, and a method setting the variable, WebObjects Builder creates this code for you:

protected String myVar;
public String myVar() {
    return myVar;
}
public void setMyVar(String newMyVar) {
    myVar = newMyVar;
}


Adding an Action

To add an action to the component:

  • Choose Add Action from the Edit Source pop-up menu at the bottom of the window

To add a key to the application or session:

  • In the object browser, select application or session, Control-click on the second column, and choose Add Action.

When you choose Add Action, a dialog appears that lets you choose the name of the action and the type of page it returns.

For example, if you create an action named myAction that returns a page of type ThatPage, WebObjects Builder creates this code for you:

public ThatPage myAction()
{
    ThatPage nextPage =
         (ThatPage)pageWithName("ThatPage");
    // Initialize your component here

    return nextPage;
}


Deleting and Renaming a Key or Action

To delete a key or action:

  • In the object browser, Control-click the name and choose Delete.

    Any variables and methods associated with the key or action are deleted from the source file. To restore the deleted key or action, chose Edit > Undo.

To rename a key or action:

  • In the object browser, Control-click the name and choose Rename.

    Any variables and methods associated with the key or action are renamed in the source file. To restore the deleted key or action, chose Edit > Undo.
