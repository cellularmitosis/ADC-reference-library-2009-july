Interface Description

The PropertyListEncoding interface declares two methods that read and write objects to property lists-a dictionary containing only property list data types (that is, next.util.ImmutableHashtables, java.lang.Strings, next.util.ImmutableVectors, and next.util.ImmutableBytes).

Classes that implement this interface must also provide a constructor that creates objects from a property list and an owner:

public ClassName(next.util.ImmutableHashtable propertyList, java.lang.Object owner)

Objects created with a constructor of this type is initialized from propertyList. The owner argument is optional and should be used only by objects requiring a reference to their owner. The newly created object isn't considered fully functional until it receives an awakeWithPropertyList message, which finishes initializing the object. The awakeWithPropertyList invocation should be deferred until after all of the objects identified in propertyList have been created.

The method encodeIntoPropertyList is responsible for encoding the receiver into a property list for later restoration.

This interface is used to read and write modeling objects (Model, Entity, Attribute, and so on) to a model file.

Instance Methods

awakeWithPropertyList

public abstract void awakeWithPropertyList(next.util.ImmutableHashtable propertyList);

Finishes initializing the receiver from propertyList, which must have been created with a constructor of the form:

public ClassName(next.util.ImmutableHashtable propertyList, java.lang.Object owner)

This method is responsible for restoring references to other objects. Consequently, it should not be invoked until all other objects that the receiver might reference have been created from propertyList.

encodeIntoPropertyList

public abstract void encodeIntoPropertyList(next.util.MutableHashtable propertyList);

Returns the receiver as a property list.
