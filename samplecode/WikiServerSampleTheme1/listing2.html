/* Define a simple property with a standard CSS rule. */
html, body { background:#ff6; }

/*
If you need to override a property that has already been defined in the 
original theme or additional 'required' CSS files, you may need to pay
attention to the rules of CSS specificity. For example:

The 'required' CSS already specifies the 'sidebar' column on the right:

.grouphome .columnprimary, .search .columnprimary { float:left; }
.grouphome .columnsecondary, .search .columnsecondary { float:right; }

So because that 'required' CSS is linked later than this file,  you'll 
need to override the rule with a higher specificity, a.k.a., a more 
specific selector.

*/

/* Notice the extra 'div' selectors here which give these a higher 'specificity' 
and allow these rules to override the conflicting ones in the 'required' CSS. */
.grouphome div.columnprimary, .search div.columnprimary { float:right; }
.grouphome div.columnsecondary, .search div.columnsecondary { float:left; }


/*
Alternately, you could use the !important directive like a trump card, 
but be careful not to use this often, or it could make debugging difficult.

.grouphome .columnprimary, .search .columnprimary { float:right !important; }
.grouphome .columnsecondary, .search .columnsecondary { float:left !important; }

*/
