QualifierSQLGeneration

Implemented By:
SQLQualifier

Interface Description

The QualifierSQLGeneration interface declares two methods that are implemented by SQLQualifier to qualify fetches from a database. One of the methods, schemaBasedQualifierWithRootEntity, is used to provide a qualifier suitable for evaluation by a database from a qualifier suitable for in-memory evaluation. The other method, sqlStringForSQLExpression, is used by SQLExpression to generate WHERE clauses for SQL statements.

Instance Methods

schemaBasedQualifierWithRootEntity

public abstract next.eo.Qualifier schemaBasedQualifierWithRootEntity(next.eo.Entity anEntity)

Returns a qualifier based on the receiver suitable for evaluation by a database (as opposed to in-memory evaluation). Invoked by a DatabaseChannel object before it uses its AdaptorChannel to perform a database operation.

See also: selectObjectsWithFetchSpecification (DatabaseChannel), schemaBasedQualifier (Entity)

sqlStringForSQLExpression

public abstract java.lang.String sqlStringForSQLExpression(next.eo.SQLExpression anSQLExpression)

Returns a SQL statement suitable for inclusion in a WHERE clause. Invoked from an SQLExpression while it's preparing a SELECT, UPDATE, or DELETE statement.

See also: whereClauseString (SQLExpression)
