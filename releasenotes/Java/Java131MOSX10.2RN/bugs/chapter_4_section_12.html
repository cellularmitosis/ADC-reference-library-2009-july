Java Printing

Radar #2816583

If the user chooses a scale in the Page Setup dialog, it isn't reflected in the Graphics2D given to the Java2 Printing API.

Description:

If the user chooses a scale in the Page Setup dialog, it should be reflected in the Graphics2D given to the Java2 Printing API. It prints correctly, but a java developer won't be able to determine that the user has scaled printing and perhaps print a different result.

Workaround:

Don't scale printing or don't have a java program rely upon knowing the scale things are being printed at.


Radar #2851380

Custom Margins not retained by PageFormat.

Description:

Setting custom page formats (margins) by calling Paper.setImageableX() (and Y()) or setImageableArea() does not have any affect on actual print preview or job from Java.

Workaround:

This has been fixed.


Radar #2869662

The pageDialog function always returns the input object.

Description:

After installing Java 1.3.1 Update 1 the pageDialog function (java.awt.print.PrinterJob class) always returns the input object regardless of the user's action and the user action is not able to be detected.

Workaround:

This has been fixed.


Radar #2871475

Java printing broken.

Description:

When printing Aqua Swing JTables using the standard Java printing API, it's been reported that an application causes a VM crash, accompanied by a malloc error message.

Workaround:

This has been fixed.
