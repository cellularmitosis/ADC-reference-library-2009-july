What is a 'scsz' resource in Java?

Q: What is an 'scsz' resource, and why do I need one in order to provide scripting support for my Java application?

A: The 'scsz' resource is a scripting size resource that provides information about an application's capabilities for use by scripting components. In AppleScript for Java, the scripting size resource determines whether dynamic terminology should be generated using the Java introspector, or whether the 'aete' should be used.

As an application developer, you can turn off dynamic generation of the 'aete' (and thus freezing your terminology) by editing the 'scsz'.

The following configurations are possible:

'scsz' present high bit set result
YES YES automatic generation on
YES NO automatic generation off
NO   automatic generation off

The default 'scsz' resource that comes with the MRJ SDK, has the high bit set. We recommend that you use this configuration during the development of your application, and then remove the resource once your 'aete' is finalized.
