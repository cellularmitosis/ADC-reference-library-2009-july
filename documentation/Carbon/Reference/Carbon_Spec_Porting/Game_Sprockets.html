Manager: Game Sprockets


The functions contained in InputSprocket.h, NetSprocket.h, and SoundSprocket.h are compatible with Carbon applications on Mac OS 8 and 9, but these functions are not available through CarbonLib on Mac OS 8 and 9, nor are they available on Mac OS X in any fashion. Apple will support a subset of the DrawSprocket APIs on Mac OS X and Mac OS 8 and 9, but these will be available only through DrawSprocketLib -- that is, not through any Carbon libraries or frameworks.

The header files InputSprocket.h, NetSprocket.h, and SoundSprocket.h contain comments describing how to use these files when building a Carbon application. Although the APIs in these header files are not available in Carbon, they are compatible with CarbonLib on Mac OS 8 and 9 and may be weak linked to avoid crashing when running on Mac OS X.

To assist you in porting your existing Mac OS 8 or 9 application to Mac OS X, Apple will support a subset of the DrawSprocket APIs for Mac OS X. This framework will not be linked into the Carbon framework. Instead, applications on both the Mac OS X and Mac OS 8 and 9 platforms should link against the CFM library DrawSprocketLib.

For detailed information on the use of GameSprocket APIs on Mac OS 8 and 9, see

http://developer.apple.com/techpubs/macos8/MultimediaGraphics/GameSprockets/applegamesprockets.html

As soon as it is available, information about using the DrawSprocket framework on Mac OS X will be published at

http://developer.apple.com/techpubs/macosx/macosx.html

Note that Apple encourages new application development on Mac OS X to use new, X-specific APIs currently under development.
