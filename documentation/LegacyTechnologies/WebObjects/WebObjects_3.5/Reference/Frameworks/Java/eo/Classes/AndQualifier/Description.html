Class Description

AndQualifier is a subclass of Qualifier that contains multiple qualifiers. AndQualifier implements the QualifierEvaluation interface, which defines the method evaluateWithObject for in-memory evaluation. When an AndQualifier object receives an evaluateWithObject message, it evaluates each of its qualifiers until one of them returns false. If one of its qualifiers returns false, the AndQualifier object returns false immediately. If all of its qualifiers return true, the AndQualifier object returns true.

Creates a new AndQualifier. If qualifiers is provided, the new AndQualifier is initialized with the Qualifier objects in qualifiers.

If aCoder is provided, the new AndQualifier is decoded from aCoder. This constructor, together with encodeWithCoder, provide the basis for archiving and distributing qualifiers. aCoder is specified when a AndQualifier is being read from an archive file; you never specify this constructor directly. For more information, see the next.util.Coding interface specification in the Foundation Framework Reference.

evaluateWithObject

QualifierEvaluation Interface

public boolean evaluateWithObject(java.lang.Object anObject)

Returns true if anObject satisfies the qualifier, false otherwise. When an AndQualifier object receives an evaluateWithObject message, it evaluates each of its qualifiers until one of them returns false. If any of its qualifiers returns false, the AndQualifier object returns false immediately. If all of its qualifiers return true, the object returns true. This method can throw one of several possible exceptions if an error occurs. If your application allows users to construct arbitrary qualifiers (such as through a user interface), you may want to write code to catch any exceptions and properly respond to errors (for example, by displaying a panel saying that the user typed a poorly formed qualifier).
